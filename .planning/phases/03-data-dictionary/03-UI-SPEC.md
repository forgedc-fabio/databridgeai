## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, chip gaps in domain tags |
| sm | 8px | Compact element spacing, table cell padding, form field gaps |
| md | 16px | Default element spacing, card body padding, tab content margin |
| lg | 24px | Section padding within side panel, nested dialog body padding |
| xl | 32px | Layout gaps between major sections, page horizontal padding |
| 2xl | 48px | Major section breaks, empty state vertical spacing |
| 3xl | 64px | Page-level top/bottom spacing |

Exceptions: Force graph container uses `h-[600px]` fixed height (consistent with Phase 2 graph container). Presentation graph view uses `calc(100vh - 64px)` for full-viewport canvas. Match table CSV upload drop zone uses `min-h-[200px]`.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `oklch(1 0 0)` / `--background` | Page background, main content area |
| Secondary (30%) | `oklch(0.97 0 0)` / `--secondary` | Cards, sidebar, tab bar background, table header, sheet overlay, domain group header rows |
| Accent (10%) | `oklch(0.205 0 0)` / `--primary` | See reserved list below |
| Destructive | `oklch(0.577 0.245 27.325)` / `--destructive` | Delete field button, delete domain button, destructive confirmation dialog primary action |

Accent reserved for:
- "Create Field" primary action button
- "Add Domain" primary action button
- "Publish Version" primary action button
- Active tab indicator (underline)
- Version dropdown selector active state
- Graph export action buttons (secondary variant)

**Domain node colours (force graph):**

These are the data dictionary domain colours for graph nodes. Each domain gets a distinct colour to differentiate categories visually.

| Domain Index | Colour | Hex | Usage |
|--------------|--------|-----|-------|
| Domain 1 | Blue | `#3b82f6` | First user-created domain |
| Domain 2 | Emerald | `#10b981` | Second user-created domain |
| Domain 3 | Amber | `#f59e0b` | Third user-created domain |
| Domain 4 | Violet | `#8b5cf6` | Fourth user-created domain |
| Domain 5 | Rose | `#f43f5e` | Fifth user-created domain |
| Domain 6 | Cyan | `#06b6d4` | Sixth user-created domain |
| Domain 7 | Orange | `#f97316` | Seventh user-created domain |
| Domain 8+ | Slate | `#64748b` | Overflow domains beyond 7 |
| Unassigned | Grey | `#6b7280` | Fields not assigned to any domain |

Colour assignment: domains are assigned colours in their display order (position 1 gets Blue, position 2 gets Emerald, etc.). This palette is stored as a `DOMAIN_COLOUR_PALETTE` constant array.

**Value type indicator colours (badges in DataTable):**

| Value Type | Badge Variant | Reasoning |
|------------|---------------|-----------|
| Text | `outline` (default) | Most common, understated |
| Long Text | `outline` (default) | Similar to Text |
| Descriptive Text | `outline` (default) | Similar to Text |
| Picklist | `secondary` | Distinct — has sub-entity (values) |
| Concatenated | `secondary` | Distinct — has sub-entity (field references) |
| Number | `outline` (default) | Simple type |

Source: Phase 02 UI-SPEC colour palette (extended for domain count), `globals.css` existing `:root` custom properties

### Fields Tab (DD-01, DD-02, DD-03)

**Focal point:** The DataTable with expandable domain groups is the primary focal element. It occupies the full content width below the tab bar. The "Create Field" primary action button (top-right, accent colour) serves as the secondary focal point.

| Component | Detail |
|-----------|--------|
| DataTable | shadcn table + `@tanstack/react-table` with grouping |
| Grouping | Expandable domain header rows — click to collapse/expand fields within that domain. Domain header shows: domain colour dot (12px circle), domain name (semibold), field count badge |
| Unassigned group | Fields not in any domain appear under "Unassigned" group at bottom (grey dot, italic name) |
| Multi-domain fields | Fields belonging to multiple domains appear under each domain group |
| Columns | Field Name, Value Type (badge), Tagging Method, Domains (badge chips, max 3 visible + "+N" overflow), Updated (relative time) |
| Default sort | Field Name ascending within each domain group; domain groups in user-defined display order |
| Row click | Opens side panel (Sheet) with field edit form |
| Row actions | DropdownMenu: "Edit", "Delete" (destructive) |
| Create button | "Create Field" — primary variant, top-right above table, opens same Sheet |
| Empty state | Displayed when zero fields exist (see Copywriting Contract below) |
| Skeleton | shadcn `Skeleton` rows while data loads |

### Field Edit Panel (DD-02, DD-03)

| Component | Detail |
|-----------|--------|
| Container | shadcn `Sheet` — slides in from right, 480px wide |
| Title | "Create Field" or "Edit Field" |
| Description | "Define a new dictionary field." or "Update the field properties below." |

**Form fields (in order):**

| Field | Component | Validation | Notes |
|-------|-----------|------------|-------|
| Field Name | `Input` | Required, Title Case enforced on blur (capitalise every word) | Placeholder: "e.g. Brand Name" |
| Field Definition | `textarea` (3 rows) | Optional | Placeholder: "Describe what this field represents. This is sent to the AI for classification context." |
| Value Type | `Select` | Required | Options: Text, Long Text, Descriptive Text, Picklist, Concatenated, Number |
| Tagging Method | `Select` | Required, default "AI Inferred" | Options: Sourced, AI Inferred, System |
| Domain Assignment | Badge/chip row with `+` button | Optional | Click to add domains from dropdown; click X on chip to remove |
| AI Instruction | `textarea` (2 rows) | Optional | Placeholder: "Additional rules for the AI beyond the field definition." |
| Controlled? | Checkbox | Visible only when Value Type = Picklist AND a match table exists for this field | Label: "Controlled by match table" |

**Conditional sections (below main form, within same panel):**

| Condition | UI Element | Behaviour |
|-----------|-----------|-----------|
| Value Type = Picklist | "Manage Values" button | Opens nested dialog over the side panel. Button shows count: "Manage Values ({N})" |
| Value Type = Concatenated | "Configure Fields" button | Opens nested dialog for ordered field selection. Button shows preview: "{Field1} \| {Field2}" |

**Panel actions (footer):**

| Action | Variant | Label |
|--------|---------|-------|
| Save | Primary | "Save Field" |
| Cancel | Ghost | "Discard Changes" |

### Match Table Upload (DD-02 sub-feature)

| Component | Detail |
|-----------|--------|
| Trigger | When a Picklist field has Controlled? = TRUE but no match table exists, show inline alert: "This field requires a match table. Upload a CSV to define cascading values." with "Upload CSV" button. |
| Upload dialog | shadcn `Dialog` — 600px wide |
| Title | "Upload Match Table" |
| Content | Drag-and-drop zone (`min-h-[200px]`, dashed border, `border-muted-foreground/30`) with text: "Drag a CSV file here, or click to browse." File input hidden behind zone. |
| Preview | After upload: show first 5 rows in a read-only preview table. Column headers auto-detected from CSV first row. |
| Actions | "Import" (primary, enabled after preview), "Cancel" (ghost) |
| CSV format | Requires headers. Multi-column hierarchy: each column represents a level (e.g., Engagement Type > Media Type > Channel > Sub-Channel). |
| v1 scope | CSV upload only. No inline editing of match table data. To update, re-upload CSV. |

### Versioning UI (DD-05)

| Component | Detail |
|-----------|--------|
| Version dropdown | In page header, right of title. Shows "v{N}" with optional label. Dropdown lists all versions, most recent first. |
| Current state | Top of dropdown: "Current Draft" if unpublished changes exist, or "v{N} (current)" if up to date |
| Publish action | "Publish Version" button at top of dropdown. Opens dialog: "Publish Dictionary Version" with optional label input and confirmation. |
| Published versions | Listed in dropdown. Click to view read-only snapshot. |

### Visualisation Tab (DD-07, DD-08, DD-09)

| Component | Detail |
|-----------|--------|
| Toggle | Segmented control (two buttons) top-right of tab content: "Graph" (default) / "Tree" |
| Graph view | `react-force-graph-2d` |
| Container height | `h-[600px]` within tab content area |

**Graph View:**

| Property | Value |
|----------|-------|
| Renderer | `react-force-graph-2d` |
| Node structure | 2-tier: domain nodes (large, 20px radius) + field nodes (small, 8px radius) |
| Edges | Domain -> Field (belongs-to). Field -> Field (concatenated references, dashed line). |
| Node colour | Domain nodes: colour from `DOMAIN_COLOUR_PALETTE` by display order. Field nodes: same colour as their primary domain, 60% opacity. Unassigned fields: grey. |
| Node label | Domain nodes: domain name (12px, white on colour). Field nodes: field name (10px, foreground colour, positioned below node). |
| Interactions | Click node: show tooltip with details. Hover: highlight connected nodes, dim others. Zoom + pan. |
| Controls bar | Domain filter (Select), Search (Input with field name autocomplete), Zoom to Fit (Button) |

**Tree View:**

| Property | Value |
|----------|-------|
| Format | Expandable tree using Collapsible components |
| Structure | Root level: domains (colour dot + name + field count). Expand: fields within that domain. Expand field: picklist values or concatenated field list (if applicable). |
| Interactions | Click domain: expand/collapse. Click field: opens field edit panel (same Sheet). |
| Unassigned | "Unassigned" node at bottom for orphan fields |
| Scroll | `ScrollArea` wrapping tree if content exceeds viewport |

## Accessibility Notes

| Concern | Treatment |
|---------|-----------|
| Tab navigation | Radix Tabs provides keyboard arrow-key navigation between tabs |
| Sheet focus trap | Radix Sheet traps focus within panel when open; Escape to close |
| Nested Dialog | Radix Dialog traps focus within dialog when open over Sheet; Escape closes dialog first, then Sheet |
| AlertDialog | Radix AlertDialog provides focus management and Escape/Enter keyboard support |
| Drag reorder | GripVertical handle is keyboard-focusable; Space/Enter activates drag mode; Arrow keys move position; Escape cancels. `aria-grabbed` and `aria-dropeffect` attributes. |
| Graph | Force graph is not keyboard-navigable — primary data access is via DataTable and Tree view (both keyboard-accessible). Graph is supplementary visualisation. |
| Colour contrast | Domain colours on graph nodes use white text for large nodes; all colours meet WCAG AA on their backgrounds |
| ARIA | Disabled nav items use `aria-disabled="true"` (established in Phase 01 sidebar). Collapsible domain groups use `aria-expanded`. |
| Title Case | Auto-correction happens on blur — screen readers announce the corrected value |
