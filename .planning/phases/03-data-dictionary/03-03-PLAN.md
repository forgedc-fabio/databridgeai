Functions:

`getPicklistValues(fieldId: string)`:
- Return type: `Promise<{ data: DictionaryPicklistValue[]; error?: undefined } | { data?: undefined; error: string }>`
- Query: `supabase.from("dictionary_picklist_values").select("*").eq("field_id", fieldId).order("display_order", { ascending: true })`

`savePicklistValues(fieldId: string, values: PicklistValueInput[])`:
- Auth check + tenant lookup
- Strategy: delete all existing values for this field_id, then insert new ones with display_order = index
- Delete: `supabase.from("dictionary_picklist_values").delete().eq("field_id", fieldId)`
- Insert: `supabase.from("dictionary_picklist_values").insert(values.map((v, i) => ({ field_id: fieldId, tenant_id, value: v.value, definition: v.definition ?? null, display_order: i })))`
- Handle empty values array (just delete, no insert)
- `revalidatePath("/dictionary")`

`getConcatenatedRefs(fieldId: string)`:
- Return type: `Promise<{ data: DictionaryConcatenatedRef[]; error?: undefined } | { data?: undefined; error: string }>`
- Query: `supabase.from("dictionary_concatenated_refs").select("*").eq("field_id", fieldId).order("position", { ascending: true })`

`saveConcatenatedRefs(fieldId: string, refs: ConcatenatedRefInput[])`:
- Auth check + tenant lookup
- Strategy: delete all existing refs for this field_id, then insert new ones
- Delete: `supabase.from("dictionary_concatenated_refs").delete().eq("field_id", fieldId)`
- Insert: `supabase.from("dictionary_concatenated_refs").insert(refs.map(r => ({ field_id: fieldId, referenced_field_id: r.referencedFieldId, tenant_id, position: r.position })))`
- Filter out self-references: `refs.filter(r => r.referencedFieldId !== fieldId)`
- `revalidatePath("/dictionary")`

"use server";

import { revalidatePath } from "next/cache";

Functions:

`getDictionaryVersions()`:
- Return type: `Promise<{ data: Pick<DictionaryVersion, "id" | "version_number" | "label" | "published_at">[]; error?: undefined } | ...>`
- Query: `supabase.from("dictionary_versions").select("id, version_number, label, published_at").order("version_number", { ascending: false })`
- Note: do NOT select snapshot in the list query (large JSONB)

`publishDictionaryVersion(label?: string)`:
- Auth check + tenant lookup
- Step 1: Get next version number: `supabase.from("dictionary_versions").select("version_number").eq("tenant_id", tenantId).order("version_number", { ascending: false }).limit(1)`. Next = (first?.version_number ?? 0) + 1.
- Step 2: Build snapshot by querying all current dictionary data:
  - Domains: `supabase.from("dictionary_domains").select("id, name, description, domain_area, owner, display_order").order("display_order")`
  - Fields: `supabase.from("dictionary_fields").select("*").order("field_name")`
  - Field-domain assignments: `supabase.from("dictionary_field_domains").select("field_id, domain_id")`
  - Picklist values: `supabase.from("dictionary_picklist_values").select("field_id, value, definition").order("display_order")`
  - Concatenated refs: `supabase.from("dictionary_concatenated_refs").select("field_id, referenced_field_id").order("position")`
- Step 3: Assemble `DictionarySnapshot` object:
  - `snapshot.domains` = domain rows
  - `snapshot.fields` = field rows with `domain_ids` (from junction), `picklist_values` (from picklist table, grouped by field_id), `concatenated_field_ids` (from concat table, grouped by field_id)
- Step 4: Insert version: `supabase.from("dictionary_versions").insert({ tenant_id: tenantId, version_number: nextNumber, label: label ?? null, snapshot, published_by: user.id })`
- `revalidatePath("/dictionary")`

`getDictionaryVersionSnapshot(versionId: string)`:
- Return type: `Promise<{ data: DictionaryVersion; error?: undefined } | ...>`
- Query: `supabase.from("dictionary_versions").select("*").eq("id", versionId).single()`

`computeVersionDiff(snapshotA: DictionarySnapshot, snapshotB: DictionarySnapshot)`:
- Pure function (not a server action — no `"use server"` needed, but keep in this file for colocation). Export it directly.
- Compare by field id:
  - `added`: fields in B not in A (by id)
  - `removed`: fields in A not in B (by id)
  - `changed`: fields in both, but with different properties. For each changed field, list which properties changed (e.g., "field_name", "value_type", "picklist_values count: 3 -> 5")
- Domain changes:
  - `domainChanges.added`: domains in B not in A
  - `domainChanges.removed`: domains in A not in B
- Return `VersionDiffResult`
