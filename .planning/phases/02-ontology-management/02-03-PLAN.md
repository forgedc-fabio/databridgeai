**File 1: `src/features/ontology/lib/validators.ts`**

Export `detectCircularHierarchy` function:

```typescript
/**
 * Detects circular hierarchies in "is-a" relationships using DFS.
 * Call BEFORE inserting a new "is-a" relationship.
 *
 * @param sourceClassId - The class that would be the subclass (child)
 * @param targetClassId - The class that would be the superclass (parent)
 * @param existingRelationships - All current "is-a" relationships
 * @param classNameMap - Map of class ID to class name (for error messages)
 * @returns { circular: false } or { circular: true, path: string[] }
 */
export function detectCircularHierarchy(
  sourceClassId: string,
  targetClassId: string,
  existingRelationships: Array<{ source_class_id: string; target_class_id: string }>,
  classNameMap: Map<string, string>
): { circular: false } | { circular: true; path: string[] }
```

Implementation: Build an adjacency list from `existingRelationships` (source -> target for "is-a" means source is subclass of target). Add the proposed edge (sourceClassId -> targetClassId). Run DFS from targetClassId following "is-a" edges upward. If DFS reaches sourceClassId, a cycle exists. Return the path as human-readable class names using classNameMap.

The DFS should:
1. Start at `targetClassId`
2. Follow all outgoing "is-a" edges (where the node is source_class_id, target_class_id is next)
3. Track visited set and current path
4. If it reaches `sourceClassId`, return `{ circular: true, path: [names along the cycle] }`
5. If exhausted without cycle, return `{ circular: false }`
