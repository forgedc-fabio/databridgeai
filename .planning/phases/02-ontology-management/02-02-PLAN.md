**File 1: `src/features/ontology/actions/class-actions.ts`**

Create server actions file with `"use server"` directive. Import `createClient` from `@/lib/supabase/server`, `revalidatePath` from `next/cache`, and `OntologyClassInput` from `../types/ontology`.

Four exported async functions:

1. `getOntologyClasses()` — Fetches all classes for the current tenant. Creates Supabase client, calls `supabase.from("ontology_classes").select("*").order("name", { ascending: true })`. Returns `{ data: OntologyClass[] }` or `{ error: string }`. RLS handles tenant scoping automatically.

2. `createOntologyClass(input: OntologyClassInput)` — Gets user via `supabase.auth.getUser()`, looks up `tenant_id` from `user_profiles` where `user_id = user.id` (single), then inserts into `ontology_classes` with fields mapped: `name: input.name, description: input.description ?? null, domain_group: input.domainGroup ?? null, colour: input.colour ?? '#6366f1', icon_tag: input.iconTag ?? null, custom_attributes: input.customAttributes ?? []`. On error code `23505` return `{ error: "A class with this name already exists. Choose a different name." }`. On success, call `revalidatePath("/ontology")` and return `{ data }`.

3. `updateOntologyClass(id: string, input: OntologyClassInput)` — Similar auth + tenant check. Updates the class where `id = id`, mapping same fields as create. Handles `23505` duplicate name error. Calls `revalidatePath("/ontology")`.

4. `deleteOntologyClass(id: string)` — Auth check. Before deleting, query `ontology_relationships` where `source_class_id = id` or `target_class_id = id` to get count. Delete the class (cascade will remove relationships). Call `revalidatePath("/ontology")`. Return `{ data: { deletedRelationships: count } }`.

Also add a helper: `getRelationshipCountForClass(classId: string)` — returns the count of relationships involving this class (for the delete confirmation dialog).
