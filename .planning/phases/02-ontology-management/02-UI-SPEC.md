## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, tight element pairs |
| sm | 8px | Compact element spacing, table cell padding, form field gaps |
| md | 16px | Default element spacing, card body padding, tab content margin |
| lg | 24px | Section padding within panels, side panel body padding |
| xl | 32px | Layout gaps between major sections, page horizontal padding |
| 2xl | 48px | Major section breaks, empty state vertical spacing |
| 3xl | 64px | Page-level top/bottom spacing |

Exceptions: Cytoscape graph container uses `h-[600px]` fixed height to ensure rendering (Cytoscape requires explicit container dimensions — see RESEARCH.md Pitfall 5). Presentation graph view uses `calc(100vh - 64px)` for full-viewport canvas.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `oklch(1 0 0)` / `--background` | Page background, main content area |
| Secondary (30%) | `oklch(0.97 0 0)` / `--secondary` | Cards, sidebar, tab bar background, table header, sheet overlay |
| Accent (10%) | `oklch(0.205 0 0)` / `--primary` | See reserved list below |

- Graph export action buttons (secondary variant)

## Component Inventory

### Page Layout

| Component | Location | Behaviour |
|-----------|----------|-----------|
| Ontology page | `/ontology` | Horizontal tab bar at top; three tabs: Class List, Relationship Editor, Visual Graph |
| Presentation view | `/ontology/visualisation` | Full-page read-only graph; no edit controls, larger canvas, shareable URL |

### Tab Bar (ONT-06)

| Property | Value |
|----------|-------|
| Component | shadcn `Tabs` (Radix `@radix-ui/react-tabs`) |
| Orientation | Horizontal, top of page below page heading |
| Tabs | "Classes" (default), "Relationships", "Graph" |
| Icons | `List` (Classes), `ArrowLeftRight` (Relationships), `Network` (Graph) — all from Lucide |
| Active indicator | Underline style, using `--primary` colour |
| Tab content | Full width, fills remaining viewport height below tab bar |

### Class List View (ONT-01, ONT-02)

**Focal point:** The DataTable is the primary focal element. It occupies the full content width below the tab bar and draws the eye via data density contrast against the white page background. The "Create Class" primary action button (top-right, accent colour) serves as the secondary focal point for empty and sparse states.

| Component | Detail |
|-----------|--------|
| DataTable | shadcn table + `@tanstack/react-table` |
| Columns | Name, Domain Group, Description (truncated), Properties count, Updated |
| Default sort | Name ascending |
| Row click | Opens side panel (Sheet) with class edit form |
| Create button | "Create Class" — primary variant, top-right above table, opens same Sheet |
| Empty state | Displayed when zero classes exist (see Copywriting Contract below) |
| Skeleton | shadcn `Skeleton` rows while data loads |

### Class Edit Panel (ONT-02)

| Component | Detail |
|-----------|--------|
| Container | shadcn `Sheet` — slides in from right, 480px wide |
| Form fields | Name (Input, required), Description (Textarea), Domain Group (Select with options: Clinical, Commercial, Regulatory, Medical, Manufacturing), Colour (colour picker or Select), Icon Tag (Input), Custom Attributes (dynamic key-value list) |
| Custom attributes | Repeatable row: Key (Input) + Type (Select: text, number, boolean, enum) + Value (Input) — add/remove buttons |
| Actions | "Save Class" (primary), "Discard Changes" (ghost/outline) — bottom of panel |
| Validation | Name required, unique within tenant; inline error messages below fields |

### Relationship Editor View (ONT-03, ONT-04)

| Component | Detail |
|-----------|--------|
| DataTable | shadcn table + `@tanstack/react-table` |
| Columns | Source Class, Relationship Type, Target Class, Created |
| Filters | Filter by class (Select), filter by relationship type (Select) — above table |
| Create button | "Add Relationship" — primary variant, top-right above table |
| Add/Edit form | shadcn `Dialog` — Source Class (Select), Relationship Type (Select with fixed types + custom), Target Class (Select) |
| Relationship types | Fixed: is-a, has-part, related-to, depends-on. Custom: user-defined via inline creation in the type Select dropdown (type to create new) |
| Delete | Row action via `DropdownMenu` — "Delete" option with destructive styling |
| Circular hierarchy | Validated on create/update — "is-a" relationships checked via server-side DFS; rejected with inline error: "Cannot create circular hierarchy: [path]" |

### Visual Graph View (ONT-07, ONT-09)

| Component | Detail |
|-----------|--------|
| Graph renderer | `react-cytoscapejs` wrapping `cytoscape`, loaded via `next/dynamic` with `ssr: false` |
| Layout | `cytoscape-dagre` — hierarchical top-down (`rankDir: "TB"`, `nodeSep: 50`, `rankSep: 80`) |
| Container height | `h-[600px]` within tab content area |
| Node styling | 60px circles, domain-coloured background, white label text, 12px font |
| Edge styling | Bezier curves, triangle arrow, slate-400 line colour, 10px rotated labels |
| "is-a" edges | Solid line, slate-500 colour, distinguished from other relationship types |
| Interactions | Click node: navigate to Class List tab for that class. Hover: highlight neighbours, dim unconnected (opacity 0.2). Drag: reposition nodes. |
| Controls bar | Top of graph area — Domain filter (Select/multi-select), Search (Input with class name autocomplete), Zoom to Fit (Button), Expand/Collapse (Button) |
| Loading state | shadcn `Skeleton` filling graph container |

### Presentation Graph (ONT-08, ONT-10)

| Component | Detail |
|-----------|--------|
| Route | `/ontology/visualisation` |
| Canvas height | `calc(100vh - 64px)` — near full viewport |
| Controls | Minimal toolbar: Domain filter, Search, Zoom to Fit, Export PNG, Export SVG |
| No edit controls | `autoungrabify: false` (nodes draggable for positioning but no CRUD actions) |
| Export PNG | `cy.png({ output: "blob", full: true, scale: 2, bg: "#ffffff" })` — downloads via `file-saver` |
| Export SVG | `cytoscape-svg` extension — downloads via `file-saver` |
| Back navigation | "Back to Editor" link/button top-left |

### Sync Button (ONT-11, ONT-12)

| Component | Detail |
|-----------|--------|
| Position | Top-right of ontology page, next to "Create Class" button |
| Label | "Sync to Cognee" |
| Stale indicator | Amber dot badge on button when `max(updated_at) > last_synced_at` |
| Syncing state | Button shows spinner icon, label changes to "Syncing...", button disabled |
| Success | Toast (sonner): "Ontology synced to Cognee" — auto-dismiss 4 seconds |
| Error | Toast (sonner): "Sync failed: [error message]. Check Cognee connection and try again." — persistent until dismissed |
| Non-blocking | User can continue working in other tabs during sync |

### Delete Confirmation (ONT-02)

| Component | Detail |
|-----------|--------|
| Container | shadcn `AlertDialog` |
| Trigger | Delete action in class row dropdown or within edit panel |
| Content | Lists relationships that will be removed with the class |
| Actions | "Delete Class" (destructive variant), "Keep Class" (outline) |

## Interaction States

| State | Visual Treatment |
|-------|-----------------|
| Table row hover | `bg-muted` background highlight |
| Table row selected (active edit) | `bg-muted` with left border accent |
| Tab active | Underline with `--primary` colour, semibold label |
| Tab inactive | No underline, regular weight, `--muted-foreground` colour |
| Tab hover | `bg-muted` background |
| Button disabled | `opacity-50`, `pointer-events-none` |
| Graph node hover | Neighbours highlighted, non-neighbours dimmed to `opacity: 0.2` |
| Graph node selected (click) | 3px `#1e40af` border |
| Sheet open | Right slide-in with backdrop overlay |
| Form field error | Red border (`--destructive`), error text below field in `--destructive` colour at 12px |
| Sync button stale | Amber dot (`oklch(0.75 0.18 75)`) positioned top-right of button |
| Sync button syncing | Spinner icon replaces sync icon, label "Syncing...", button disabled |
| Skeleton loading | shadcn `Skeleton` pulse animation, matching content shape |

## Accessibility Notes

| Concern | Treatment |
|---------|-----------|
| Tab navigation | Radix Tabs provides keyboard arrow-key navigation between tabs |
| Sheet focus trap | Radix Sheet traps focus within panel when open; Escape to close |
| AlertDialog | Radix AlertDialog provides focus management and Escape/Enter keyboard support |
| Graph | Cytoscape graph is not keyboard-navigable — primary data access is via DataTable (keyboard-accessible). Graph is supplementary visualisation |
| Colour contrast | Domain colours on graph nodes use white text; all colours meet WCAG AA on their backgrounds |
| ARIA | Disabled nav items use `aria-disabled="true"` (established in Phase 01 sidebar) |
