## Interaction Patterns

### Login Flow
1. User opens application URL â†’ proxy.ts checks auth state â†’ redirects to `/login`
2. User enters email and password â†’ clicks "Sign in"
3. Button enters loading state (disabled, spinner or loading text)
4. **Success:** redirect to dashboard (`/`)
5. **Failure:** inline error message below form, form fields remain populated, focus returns to password field

### Sidebar Behaviour
1. **Expanded state (default):** 256px width, icons + text labels visible
2. **Collapsed state:** Icon-only, tooltips on hover for each nav item
3. **Toggle:** Click SidebarTrigger or press Cmd+B / Ctrl+B
4. **Persistence:** Collapsed/expanded state stored in cookie (shadcn/ui default)
5. **Disabled items:** Visual muting (opacity-50), no click handler, tooltip explains unavailability

### Health Status Polling
1. Dashboard mounts â†’ immediate health check request via `/api/health/cognee` route
2. Poll every 30 seconds while dashboard is visible
3. Display status dot + label for each service (Cognee, Storage)
4. On status change, update dot color and label text immediately (no page refresh)

- Main content area: padding lg (24px) on all sides

- Health status card: full width, xl (32px) margin-top below welcome card
- Health status items: stacked vertically within card, sm (8px) gap between service rows

## Responsive Behaviour

Phase 1 targets desktop usage (stated constraint: "web-first, desktop users"). Minimum supported viewport: 1024px width.

- **Below 1024px:** shadcn/ui Sidebar automatically switches to mobile sheet overlay (built-in behaviour)
- **Above 1024px:** Standard sidebar + content layout

No additional responsive breakpoints or mobile-specific layouts for Phase 1.

---

## Accessibility

- All form inputs have associated `<label>` elements (shadcn/ui `Label` component)
- Login form is keyboard-navigable (Tab between fields, Enter to submit)
- Sidebar keyboard shortcut: Cmd+B / Ctrl+B (shadcn/ui built-in)
- Health status dots use aria-label describing the status (e.g., `aria-label="Cognee status: connected"`)
- Disabled nav items have `aria-disabled="true"` and descriptive tooltip
- Color is not the sole indicator â€” all status dots are paired with text labels
